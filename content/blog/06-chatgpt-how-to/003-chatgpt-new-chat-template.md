# ChatGPT新規チャットで迷わない：添付テンプレの運用法

ChatGPT で新しいチャットを開くたびに、前回の文脈がリセットされてしまい「またゼロから説明するのか…」と感じたことはないでしょうか。

筆者もこの問題に悩んでいましたが、添付ファイルの順番と役割を整理したことで、毎回の立ち上げがスムーズになりました。

> **この記事の結論**
>
> - 新規チャットはコンテキストが空なので、添付テンプレを固定すると迷わない
> - 添付順は `context.md` → `project-context.md` → `source.md` → 依頼が基本
> - `writing-rules.md` は文体ブレが気になったときだけ再添付する
> - `context.md` を固定化すると token 効率と再現性が上がる

---

## なぜ新規チャットのたびに添付が必要なのか

ChatGPT（執筆時点）は、チャットをまたいで会話の記憶を保持しません。新規チャットを開くと、AI にとっては「はじめまして」の状態です。

このため、プロジェクトの前提・文体ルール・読者像などを毎回伝え直す必要があります。その手間を最小化するのが、**コンテキストファイルの固定添付**です。

---

## 添付ファイルの役割と順番

### 各ファイルの役割

| ファイル             | 役割                                         | 添付タイミング           |
| -------------------- | -------------------------------------------- | ------------------------ |
| `context.md`         | プロジェクト全体の前提・役割分担             | **毎回**固定             |
| `project-context.md` | サイト概要・読者・技術スタック               | **毎回**固定             |
| `source.md`          | 記事ごとの設計メモ（テーマ・問題・伝えたいこと） | 記事化するときのみ       |
| `writing-rules.md`   | 文体・Markdown・SEO・収益導線の詳細ルール    | 文体がずれたときに再添付 |

### 推奨する添付順

```
context.md → project-context.md → source.md → 依頼文
```

先に「前提となる広い情報」を渡し、最後に「今回の具体的な作業指示」を置くことで、AI が依頼の意図を正確に受け取りやすくなります。

---

## 実際の手順

1. `context.md` と `project-context.md` を最新の状態に保つ
2. 記事化したい素材は `source.md` に構造化してまとめる（daily notes の丸投げはしない）
3. 新規チャットで上記を順に添付し、最後に具体的な依頼文を書く
4. 文体がずれていると感じたら `writing-rules.md` だけ再添付する
5. ズレが大きい場合はコンテキストファイルの該当セクションだけ貼り直す

---

## token 効率と再現性を両立するコツ

`writing-rules.md` はファイルサイズが大きくなりやすいため、毎回添付すると token を消費します。文体の詳細ルールが必要なタイミング（初稿の文体確認、長文記事の本文生成など）だけに絞ると、コストと精度のバランスが取れます。

一方、`context.md` は短くまとめておくことで、毎回添付してもほぼ負担になりません。プロジェクトの役割分担・禁止事項・出力方針をここに集約しておくのがポイントです。

---

## まとめ

| やること                         | 頻度           |
| -------------------------------- | -------------- |
| `context.md` + `project-context.md` を添付 | 毎回           |
| `source.md` を添付               | 記事化するとき |
| `writing-rules.md` を添付        | 文体がずれたとき |
| 各ファイルを最新状態に保つ       | 内容が変わったとき |

添付テンプレを固定化することで、「何を渡せばいいか」で迷う時間がなくなり、依頼の質と再現性が上がります。まずは `context.md` 1ファイルから始めてみてください。
