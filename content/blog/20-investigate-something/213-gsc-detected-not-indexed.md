# 「検出 - インデックス未登録」の意味と初動対策【GSC】

Google Search Console（GSC）を確認したら「検出 - インデックス未登録」という表示があった。「エラーなのか、待てばいいのか」判断に迷う方は多いです。

結論から言うと、すぐに深刻な問題とは限りません。ただし放置していいわけでもないため、状態の意味を正しく理解したうえで初動を整えることが大切です。

> **この記事の結論**
>
> - 「検出 - インデックス未登録」は、Google が URL の存在を把握しているが、まだクロール・登録していない状態の一例
> - 新規ドメインはクロール頻度が低い傾向があるため、短期間で大きく動かそうとしない
> - サイトマップ・内部リンク・公開品質を整えたうえで、1〜3か月は推移を記録するのが現実的

---

## 「検出 - インデックス未登録」とはどういう状態か

GSC のページレポートに表示されるこのステータスは、大まかに次の状態を示しています（執筆時点）。

| 状態 | 意味 |
| ---- | ---- |
| 検出済み | Google がサイトマップや内部リンクを通じて URL の存在を把握している |
| インデックス未登録 | クロール（ページの中身の取得・解析）がまだ完了しておらず、検索データベースに登録されていない |

つまり「存在は知っているが、まだ読んでいない」状態です。エラーではなく、クロールの順番待ちに近いイメージです。

---

## なぜこの状態になるのか

主な原因は次の3つです。

1. **クロール頻度が低い**: 新規ドメインは Google からの信頼評価が蓄積されていないため、クロール頻度が低くなる傾向があります（執筆時点の一般的な傾向であり、個別の状況は異なる場合があります）
2. **サイトマップや内部リンクが未整備**: Google がページを「発見」できていても、到達経路が少ないとクロールの優先度が上がりにくい場合があります
3. **コンテンツの品質評価**: クロール対象として優先度を付ける際、品質の低いページは後回しにされる可能性があります

---

## 初動対策：まず整えるべき3点

### 1. サイトマップの送信状況を確認する

GSC の「サイトマップ」メニューから、サイトマップが送信済みかつ「成功」状態になっているかを確認します。未送信の場合は `sitemap.xml` の URL を登録してください。

### 2. 内部リンクからの到達経路を点検する

該当ページが他のページからリンクされているかを確認します。孤立したページ（どこからもリンクされていないページ）はクロールされにくいため、Hub 記事やカテゴリページから内部リンクを追加します。

### 3. 1〜3か月は推移を記録して様子を見る

新規ドメインの初期は、対策を打ってもインデックスが増えるまでに時間がかかる場合があります。週単位でインデックス済み数の推移を記録し、急激な変化がなければ焦らず記事の品質と本数を積み上げることが先決です。

---

## やってはいけないこと

- **URL 検査の「インデックス登録をリクエスト」を連打しない**: 頻繁なリクエストは効果が薄く、むしろ無駄なクロール消費になる可能性があります
- **短期間でサイト構造を大きく変える**: 変更のたびに Google の評価がリセットされるリスクがあります
- **noindex の設定漏れを見落とす**: 意図せず `noindex` が残っていると、どれだけ待ってもインデックスされません。該当ページのソースや Next.js のメタタグ設定を確認してください

---

## まとめ

| 確認・対応項目 | 優先度 |
| -------------- | ------ |
| GSC でステータスと対象 URL を確認 | 高 |
| サイトマップの送信・成功状態を確認 | 高 |
| 内部リンクからの到達経路を追加 | 中 |
| noindex の設定漏れがないか確認 | 高 |
| 1〜3か月、週次で推移を記録 | 中 |

「検出 - インデックス未登録」は、適切な環境を整えれば自然に解消することが多い状態です。まずはサイトマップと内部リンクを点検し、あとは記録しながら待つのが現実的な初動です。
