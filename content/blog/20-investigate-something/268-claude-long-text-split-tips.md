# Claudeに長い記事を貼ると途中で切れるときの分け方

Obsidianでまとめた記事素材やノートをClaudeにそのまま貼り付けて、構成や文章を整えてもらおうとしたとき、出力が途中で止まってしまった経験はないでしょうか。筆者も、見出しが10個近くある長文記事を一度に貼り付けたところ、本文が途中で切れてしまい、続きをどう依頼すればいいか迷ったことがあります。

長文を一度に処理させようとすると、出力が途中で止まる、あるいは文章のつながりが不自然になることがあります。本記事では、その対処として「見出し単位で分割して送る」というシンプルな方法を、筆者が実際に試した手順としてまとめます。

> **今日の結論**
> - 長文をそのまま貼ると、途中で出力が止まることがある
> - 見出し単位で2〜3分割し、「分割1/3」のように番号を付けて送る
> - 1ブロックずつ送り、「続きは次に貼ります」と伝えてから次を渡す
> - 最後に「全体を通して整えて」と依頼し、見出しのつながりを確認する

## なぜ長文を貼ると途中で切れるのか

Claudeに限らず、AIの出力には一度に出せる長さの上限があります。見出しが多い記事や本文が長い記事を一度に貼ると、最後まで出力されずに途中で止まってしまうことがあります。

この上限は執筆時点（2026-06-19）の実測に基づく目安であり、仕様は変更される可能性があります。最新の制限については公式ヘルプもあわせて確認してください。

## 見出し単位で分割して送る手順

1. 記事を見出し単位で2〜3分割し、各ブロックに「分割1/3」のように番号を付ける
2. 1ブロックずつ貼り付けて送り、「続きは次に貼ります」と一言添えてから次のブロックを渡す
3. 全ブロックを送り終えたら、「全体を通して整えて」と依頼し、見出し同士のつながりや表現の重複を確認してもらう

実際に送るときの文面はこのようなイメージです。

```text
分割1/3です。続きは次に貼ります。
（ここに本文の前半を貼る）
```

## 分割時に確認しておきたいポイント

| チェック項目 | 内容 |
| --- | --- |
| 分割の境目 | 段落の途中ではなく、見出し単位で区切る |
| 番号の明示 | 「分割1/3」など、何番目かを毎回伝える |
| 最後の通し読み | 全ブロック送付後に「全体を通して整えて」と依頼する |

## まとめ・次に読む

長文を一度に渡すと出力が途中で切れることがありますが、見出し単位で分割し、番号を付けて順番に送るだけで対処できます。
