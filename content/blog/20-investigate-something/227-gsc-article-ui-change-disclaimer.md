# GSC記事は表示文言の変更可能性を免責に明記する

> **この記事の結論**
>
> - Google Search Console（GSC）の表示項目・文言は、予告なく変更される可能性がある
> - GSC 系 Tips 記事は、**本文の操作手順**と**末尾の免責**の両方に「執筆時点」を明記するのが安全
> - `source.md` の note 欄に変更可能性を書いておくと、Claude 初稿時に免責が抜け落ちにくくなる

---

## なぜ GSC 記事に免責が必要なのか

GSC の画面表示は Google 側の都合でいつでも変わります。「クロール済み・インデックス未登録」のような**ステータス文言**も、過去に何度か表現が変更されています。

スクリーンショット付きで手順を書いた記事ほど、UI が変わったときに「画面と違う」という読者の混乱を招きやすくなります。免責を入れておくことは、読者への誠実さと記事の長期的な信頼性につながります。

---

## 手順：免責を抜け落とさないための運用

### 1. source.md の note 欄に記録する

記事設計メモ（source.md）の note 欄に、執筆時点でわかっている変更リスクを書きます。

```md
note: GSC表示文言は変更される可能性あり（要確認）
```

この一行があると、Claude に初稿を依頼するときに「note の内容を免責に反映してください」と指示するだけで済みます。記事ごとに都度説明する手間が省けます。

### 2. Claude 初稿の依頼時に免責反映を明示する

初稿依頼のプロンプトに、note の内容を免責へ反映する指示を含めます。

```md
<!-- 依頼例 -->
source.md の note に「GSC表示文言は変更される可能性あり」とあります。
本文の操作手順と末尾の免責に反映してください。
```

口頭で補足しなくても、note の内容がそのまま初稿に乗るようになります。

### 3. 本文の操作手順に「執筆時点の表示名」と明記する

画面名やメニュー名を書く際は、括弧書きで補足します。

```md
<!-- 書き方の例 -->
左メニューの「インデックス作成」>「ページ」を開きます（執筆時点の表示名）。
```

一言添えるだけで、UI が変わっても読者が「古い記事かもしれない」と自分で判断できるようになります。

### 4. 記事末尾に免責を追記する

記事の末尾に次のような一文を入れます。

```md
Google Search Console の表示項目・文言は変更される可能性があります。
最新の画面表示は [Google 公式ヘルプ](https://support.google.com/webmasters/) でご確認ください。
```

### 5. 公開後の変更に備えてメモを残す

UI が変わった場合に備えて、差分メモを残しておく習慣をつけると対応が早くなります。Obsidian の Daily Note や source.md の note 欄に「〇〇の表示が変わった可能性あり」と追記し、必要に応じてリライト候補に入れます。

## チェックリスト

GSC 系記事の公開前に使えるチェックリストです。

- [ ] source.md の note 欄に変更可能性を記録した
- [ ] 本文の操作手順に「執筆時点の表示名」と明記した
- [ ] 末尾の免責に「最新は公式ヘルプで確認」を追記した
- [ ] スクリーンショットのキャプションに撮影時点を入れた（画像がある場合）

---

## まとめ

GSC の UI 変更は防ぎようがありませんが、免責の書き方をパターン化しておけば対応コストは最小限に抑えられます。source.md の note 欄への一行メモと、本文・末尾の定型文がセットになると、記事数が増えても抜け漏れが起きにくくなります。
