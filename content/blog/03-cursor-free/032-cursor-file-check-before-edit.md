# Cursorで変更前に確認したいファイルの見方

Cursorで修正を始めようとしたとき、「どのファイルを直せばいいのか分からない」と手が止まってしまうことがあります。エラーメッセージを見ても、関係するファイルがすぐには分からない、というケースは初心者にはよくある悩みです。

本記事では、AIに質問する前段階として、自分の目で確認しておきたい「ファイルの見方」を3つの視点に絞って整理します。

> **今日の結論**
> - 変更前に、いま画面に開いているファイルをまず確認する
> - エラーや目的に直接関係するファイルだけに絞り込む
> - 開いたファイルの役割を1行でメモしておく
> - この3点を押さえるだけで、的外れな修正を減らせます

## なぜ「どのファイルを見るか」で迷うのか

Cursorの画面には複数のファイルがタブで開いていることが多く、慣れていないとどれが今の作業に関係するファイルなのか判断しづらいことがあります。とくに、エラーメッセージに表示されるファイル名と、実際に修正したいファイルが一致しないケースでは、関係のないファイルを開いたまま作業を始めてしまうことがあります。

原因として考えられるのは、画面に開いているファイルの「役割」を意識せずに作業を始めてしまう点です。役割を整理せずに修正に着手すると、後から「結局どのファイルを直したのか」自分でも分からなくなることがあります。

## 変更前に確認したい3つの視点

### 1. いま開いているファイルを確認する

最初に、画面上で開いているファイルのタブを一通り確認します。複数のタブが開いている場合は、今の作業に関係ないファイルが残っていないかをチェックします。関係のないファイルが開いたままだと、誤ってそのファイルを編集してしまうリスクが上がります。

### 2. エラーや目的に関係するファイルだけに絞る

エラーメッセージが出ている場合は、そのメッセージに記載されているファイル名・行番号を確認します。目的が「ある画面の表示を直したい」など機能ベースの場合は、関係しそうなファイルを開いたタブの中から絞り込みます。関係のないファイルは一度閉じておくと、画面がすっきりして判断しやすくなります。

### 3. ファイルの役割を1行でメモする

絞り込んだファイルについて、「このファイルは何をしているか」を1行でメモしておきます。たとえば「このファイルは画面の表示部分」「このファイルはデータの取得処理」のように書いておくと、後から見返したときに迷いません。AIに修正を依頼する際も、この1行メモがあると指示の精度が上がりやすくなります。

## 確認の手順（まとめ）

| 手順 | 内容 |
| --- | --- |
| 1 | 画面上で開いているファイルを確認する |
| 2 | エラーや目的に関係するファイルだけに絞る |
| 3 | 変更前に役割を1行でメモする |

この3手順は、AIに依頼する前の「自分の頭の中の整理」として機能します。手順自体はシンプルですが、抜けがちなプロセスです。

## 確認チェックリスト

| 確認項目 | 状態 |
| --- | --- |
| 関係のないファイルを開いたままにしていないか | ☐ |
| エラーメッセージのファイル名・行番号を確認したか | ☐ |
| 修正対象のファイルを1〜2個に絞れたか | ☐ |
| 各ファイルの役割を1行でメモしたか | ☐ |

作業を始める前にこのチェックリストを一通り確認すると、的外れな修正に時間を使うことを減らせます。

## まとめ・次に読む

Cursorで変更を始める前は、次の3点を意識しておくと安心です。

- いま開いているファイルを確認し、関係のないものは閉じる
- エラーや目的に直接関係するファイルだけに絞り込む
- 絞り込んだファイルの役割を1行でメモしておく
