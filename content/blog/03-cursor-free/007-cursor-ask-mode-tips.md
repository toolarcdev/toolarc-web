# Cursorで資料を先読みさせるときはAskモードを使う

Daily Note や AI ログを Cursor に読ませてから実装に進みたい——そんな場面で、モードを間違えると Cursor が意図せず実装やファイル作成まで走ってしまうことがあります。

本記事では「資料の先読み → 実装」の流れに合ったモード選択を、3ステップで整理します。

---

> **この記事の結論**
>
> - 資料を読ませるだけなら **Ask モード**が向きやすい
> - **Plan モード**は設計・実装に進みやすい傾向があり、「読むだけ」では過剰になることがある
> - モードを目的ごとに切り替えることで、意図しない先行実装を防げる

---

## Ask と Plan、何が違うのか

Cursor には大きく Ask・Plan・Agent の3モードがあります（執筆時点）。それぞれの傾向を整理すると次のとおりです。

| モード | 主な用途 | 「読むだけ」の依頼をしたとき |
|--------|----------|------------------------------|
| **Ask** | 質問・調査・確認 | 内容を読んで回答する |
| **Plan** | 設計・実装の計画立案 | 計画を立てたうえで実装に進みやすい |
| **Agent** | 自律的なコード実装・ファイル操作 | ファイルの作成・編集まで実行する |

「この資料を読んでおいて」と依頼しても、Plan モードでは読んだ後に実装や記事作成まで進むことがあります。まず内容を確認したいだけなら、Ask モードが意図に合いやすいです。

---

## 資料の先読みに Ask モードを使う手順

### ステップ 1: Ask モードで資料を添付して読み込む

Daily Note・AI ログ・`source.md` など、先に読ませたいファイルを Ask モードで添付します。

```text
（Ask モードで）
この source.md を読んで、構成のポイントを整理してください。
```

実装の依頼は一切せず、「読んで確認する」だけに絞るのがポイントです。

### ステップ 2: 読み込み結果を確認する

Cursor が返した内容を見て、ファイルの要点が正しく読み取れているかを確認します。ここで認識のズレを直しておくと、次のステップがスムーズになります。

### ステップ 3: 実装・記事化は Plan / Agent に切り替える

内容確認が終わったら、モードを切り替えて実装や記事化の依頼をします。

```text
（Plan モードに切り替えて）
先ほどの source.md をもとに、記事の構成案を作ってください。
```

`source.md` は意図したタイミングで添付する運用が向きやすいです。自動参照になっている場合は、実際に何が読み込まれているかを都度確認すると安心です。

---

## Plan / Agent に切り替えるタイミング

| タイミング | 推奨モード |
|------------|------------|
| 資料の内容確認・質問 | Ask |
| 構成案・設計の作成 | Plan |
| ファイルの作成・コード実装 | Agent |

迷ったら「自分は今、読みたいのか・設計したいのか・実装したいのか」を一度確認してからモードを選ぶと、余計な実行を防げます。

---

## まとめ

- **Ask モード**: 資料の読み込み・確認・質問に向きやすい
- **Plan モード**: 設計・実装計画に向きやすい。「読むだけ」では意図より先に進むことがある
- 3ステップ（Ask で読む → 確認 → Plan / Agent で実装）を意識すると、ワークフローが安定しやすくなります
