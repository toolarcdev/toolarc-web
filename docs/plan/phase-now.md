## 3. Phase1 — 検証（カレンダー継続中）＋ Phase2-0 先行

### 説明

比較記事を **直接現金化しない**（ChatGPT/Claude/Cursorの現金ASPが無い前提）。  
比較＝集客、周辺収益記事＝ASP、の2段導線を試し、勝ちカテゴリをデータで決める。

並行して **Phase2-0** を先行し、ASPクリックの信号を取る。売上額はまだ必須ではない。

### 主目的（v0.2 / 7月実行計画と整合）

- Cursor / ChatGPT / Claude 主力記事の整備
- 収益導線を複数記事へ設置し、GSC/GA4/ASPを測れる状態にする
- 「どの記事タイプからクリックが出るか」を確認する

### 達成条件（Phase1完了＝9月末判定の正本）

以下を **すべて** 満たす（事業計画書 v0.2 §10）:

| # | 条件 | 備考 |
|---|------|------|
| 1 | Cursor / ChatGPT / Claude の主力記事群が整備 | Hub＋スポークで辿れる |
| 2 | L2以上の検証付き比較記事が **8本以上** | 実測・比較表つき |
| 3 | 収益導線設置記事が **10本以上** | `affiliate-registry` 案件キー利用 |
| 4 | **ASPクリックが1件以上** | ASP管理画面の実測（GA4 outboundは補助） |
| 5 | GSC表示・クリックが開始時点より増加 | 週次ログで確認 |
| 6 | 反応カテゴリを言語化できる | 例: Cursor制限 / ChatGPT移行 等 |

**継続可の例外**: 売上¥0でも、上記の信号があればPhase1成功。  
**見直しトリガー**: 記事だけ増え、クリックも導線反応もない → テーマ・CTA・導線を再設計。

### カレンダーPhase2への移行（合意ルール）

- **主条件**: Phase2-0 を卒業する（下記「2-0の卒業」＝ **ASPクリック累計10**）
- 2-0卒業時点で、v0.2の10月開始を待たず **カレンダー上も Phase2 に移行**する
- Phase1 §達成条件の9月末レビューは並行実施（未達項目は Phase2 初期の宿題として残す）

### 運営ルール（方針B・Phase1後半）

- 新規公開ペースは維持してよい
- 水曜 `#6` 負債払いは **原則2単位**（Hub更新 / 昇格PR を優先）
- 新シリーズは月1・統合優先（既存ガードレール維持）
- 7月の当面KPI: ASPクリック累計の可視化、本命CV記事 **3本**（revenue-signals）

### 次Phaseへの出口

**Phase2-0 卒業 → カレンダー Phase2（2-1以降を追う）**。

## 4. Phase2 — 収益強化（円建て細分化）

### 説明

勝ちカテゴリへ集中投資し、CTA・内部リンク・リライトで **承認収益を再現可能な水準**まで上げる。  
広告・低単価Amazon等は補助。デジタル商品は Phase2-2以降で「検証」、本格拡大は Phase3寄り。

**Phase2-0** はカレンダー Phase1 中の先行トラック。卒業後に Phase2（2-1〜）へ入る。

### 収益の定義（判定に使う数字）

- **月次収益** = 各ASPの **確定（承認）報酬合計**（見込み・未承認は含めない）
- 通貨: 円。外貨は月末レートで円換算し月次メモに明記
- 補助指標: ASPクリック、outbound_click、収益記事PV、成約件数（発生／承認）
- 単価・成果条件は記事本文に転載しない（Vault案件一覧が正本）

### 中期ファネル目安（v0.2 / revenue-signals と同一）

成約 **5件/月** を中期マイルストーンとするときの仮説（CVR等は実測で更新）:

| 指標 | 月間目安 |
|------|----------:|
| GSC表示 | 150,000〜220,000 |
| GSCクリック | 6,000〜8,000 |
| 収益記事PV | 5,000 |
| 収益導線クリック | 500〜800 |
| ASPクリック | 250 |
| 成約 | 5 |

※ 成約5の **円換算は案件ミックス次第**。Phase2の昇格は **円（承認収益）を主**、成約件数は併記とする。

### Phase2-x 細分化

| サブ | 名称 | 月次承認収益 | 入口条件 | 達成／卒業条件 | 主な施策 |
|------|------|-------------:|----------|----------------|----------|
| **2-0** | 収益信号確認 | ¥0（クリックのみ可） | 導線設置開始／本命CV候補あり | **下記「2-0の卒業」** | 計測整備、CTR改善、2段導線の実装漏れ修正、Hub→周辺の送客 |
| **2-1** | 初収益 | **¥1〜¥9,999** | 2-0卒業（＝Phase2移行） | **いずれかの月**で承認収益 ≥¥1、および成約（発生）≥1 | CTA改善、本命3本集中、否認理由の記録 |
| **2-2** | 再現 | **¥10,000〜¥49,999** | 2-1達成 | 承認収益が帯内で **連続2か月**、または単月 **¥30,000**以上 | リライト週次、勝ちカテゴリへ新規を寄せる、Aランク案件の追加判断 |
| **2-3** | 安定萌芽 | **¥50,000〜¥149,999** | 2-2達成 | 帯内 **連続2か月**、かつ成約 **月3件以上**（承認ベース推奨） | ファネルが「成約5」に近づくよう母数（GSC/PV）を拡大、内部リンク監査 |
| **2-4** | 拡張 | **¥150,000〜¥399,999** | 2-3達成 | 帯内 **連続2か月**、収益カテゴリ **2系統以上**（例: 学習系＋業務AI系） | 案件ポートフォリオ分散、比較Hub強化、必要なら広告／教材の補助導入 |
| **2-5** | 準KGI | **¥400,000〜¥999,999** | 2-4達成 | 帯内 **連続3か月**、または単月 **¥700,000**以上 | Phase3準備（指名・SNS試験・商品MVP検討）。月収100万は複合でPhase3/4側 |

### 2-0の卒業（合意正本）

| 優先 | 条件 | 扱い |
|------|------|------|
| **主** | ASP管理画面のクリックが **累計10以上** | **これで卒業・カレンダー Phase2 へ移行** |
| 副（任意メモ） | 月次 ASPクリック ≥1 が連続 | 早期シグナル。卒業の必須条件にはしない |

卒業時の作業:

1. 本ファイルの「現行ステータス」を Phase2 / サブ 2-1 待ち に更新
2. Vault `revenue-signals` に累計クリックと卒業日を記録
3. `seo-goals.md` の Outcome に ASP/承認収益の比重を上げる（6に合わせて改訂）

### 昇格・降格ルール（2-1以降）

- 上位帯に入ったらステータスを上げる（一時的な単月スパイクはメモし、**連続条件**で卒業判定）
- 2か月連続で下限を割ったら **1段降格**
- 2-0中は売上¥0でも正式サブPhaseとして扱う

### Phase2全体の達成条件（→ Phase3）

次を満たしたら Phase3 検討に入る:

- 月次承認収益が **Phase2-4以上を2か月連続**、または **2-5到達**
- 勝ちカテゴリが文書化され、新規記事の方針が「探索」より「収穫」中心
- アフィリエイト以外の第2収益の検証計画がある（広告／デジタル商品／スポンサーのいずれか）

### Phase2の撤退・減速ライン（収益側）

- 2-0を **90日**やっても ASPクリック累計が伸びない（目安: 累計3未満） → 導線設計を停止点検（量産は維持可、収益記事の型を作り直す）
- 承認率が異常に低い／規約リスク → 当該案件停止、別案件へ振替
- GSCクリックが **4週連続で半減級** → ②優先、収益実験は縮小

## 5. Phase3 — ブランド／チャネル拡大

### 説明

SEO依存を下げ、指名検索・SNS・AEO/GEO・無料ツール検討で集客を複線化する。収益は Phase2で得た型の横展開。

### 達成条件（目安）

- 月次承認収益が Phase2-4以上を維持しつつ、**指名検索またはリピーター指標**が前四半期比で改善
- X / Instagram / Shorts のうち **1媒体で週次運用が12週継続**
- 勝ちカテゴリ向けの無料ツールまたはデジタル商品の **MVP公開（有料でなくて可）**

### 次Phase

独自サービス／スポンサー／SaaS化の本格検討（Phase4）。

---

## 6. Phase4 — 事業化

### 説明

アフィリエイトを一部に下げ、独自サービス・テンプレ・コミュニティ・スポンサー等の複合で **月収100万円** を安定軌道に乗せる。

### 達成条件（KGI）

- **暦月の事業収入合計が ¥1,000,000 以上を3か月中2か月**（税込/税抜は社内で統一）  
  ※ v0.2どおりアフィリエイト単体前提ではない
- 独自プロダクトまたは継続課金の売上比率が定義済み（数値は到達間近で確定）

## 7. 方針Bの運用固定（負債）

量産は維持する。代わりにシリーズ負債を並行消化する。

| 項目 | ルール |
|------|--------|
| 新規公開 | 日次ペース維持可（reader供給枯渇時のみテーマ優先度を上げる） |
| 水曜 `#6` | **原則2単位**（P0 Hub → P1 昇格）。同一シリーズの同時大量昇格はしない |
| 赤域（`20`≥70 かつ 昇格待ち≥8） | 公開は継続。ただし翌営業日に整理①を **追加1枠** |
| 新シリーズ | 月≤1、既存統合優先、アクティブ≤12 |

詳細手順は [`debt-paydown-workflow.md`](../ai-context/debt-paydown-workflow.md) と同期する。
