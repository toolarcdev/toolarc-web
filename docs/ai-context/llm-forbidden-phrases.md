## 禁止する例（空句・均質化の種）

記事本文で次を避ける。言い換えか削除する。

| 類型 | 例 |
|------|----|
| 空の強調 | 「重要なのは」「不可欠」「欠かせない」だけで論を進める |
| 作業の実況 | 「掘り下げる」「深掘りする」（実際の手順・根拠がないとき） |
| 曖昧な網羅語 | 「多角的」「包括的」「総合的に」だけで中身がない |
| 接続の連打 | 「さらに」「また」「加えて」を短文で連続させる |
| 英語調ダッシュ（文中の間・見出し） | `――` や同型の `——`。文中で「〔状況〕→〔問い／ラベル〕」の息継ぎをする用法、および見出し（`#`〜`######`）・`title` 内の区切り。How-to では違和感が強い。文中は句点分割・読者への問い・「そのあと」等へ。見出しは `｜`（または `：`）へ置き換える。※本文箇条書きの短い区切り `—`（単独）は本行の対象外（別判断） |
| 括弧の入れ子 | `（「…」）` / `「（…）」` / `「(…)`」 は基本的に使わない。括弧内の強調は `"…"` や `` `…` ``。文分けや「たとえば」でも可。未公開は `「準備中」` |

上記は最低限。同種の「意味のない接続・強調」も同様に避ける。

---

## 禁止しない語（誤検知に注意）

次は**禁止リストに入れない**。人間の自然な語であり、禁止すると誤検知が増える（分析書 §3.7(a) の実測反証）。

- 「最後に」
- 「まさに」
- 「体言止め」（用語・説明として使うこと自体は可。運用は [`writing-rules.md`](./writing-rules.md) の山場・区切り限定に従う）

## 誤検知注意（1行）

禁止語リストの丸写しはむしろ有害。AI臭は語彙よりリズム（文長・段落の均質さ）に出る。
